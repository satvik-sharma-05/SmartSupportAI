# 🚀 Deployment Checklist

Use this checklist to deploy SmartSupport AI for free.

## ✅ Pre-Deployment

- [ ] Model file downloaded and placed in `models/smartsupport_model/model.pt`
- [ ] Backend runs locally (`uvicorn app.main:app --reload`)
- [ ] Frontend runs locally (`npm run dev`)
- [ ] Test predictions work
- [ ] Git repository created
- [ ] Code pushed to GitHub

## 🔧 Backend Deployment (Render)

- [ ] Create Render account
- [ ] Create new Web Service
- [ ] Connect GitHub repository
- [ ] Set build command: `pip install -r requirements.txt`
- [ ] Set start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- [ ] Add environment variable: `SECRET_KEY=<random-string>`
- [ ] Deploy and wait for build
- [ ] Test health endpoint: `https://your-app.onrender.com/health`
- [ ] Copy backend URL

## 🎨 Frontend Deployment (Vercel)

- [ ] Create Vercel account
- [ ] Import GitHub repository
- [ ] Set root directory: `frontend`
- [ ] Set framework: Vite
- [ ] Add environment variable: `VITE_API_URL=<your-backend-url>`
- [ ] Deploy
- [ ] Test frontend loads
- [ ] Test predictions work end-to-end

## 🗄️ Database (Optional - MongoDB Atlas)

- [ ] Create MongoDB Atlas account
- [ ] Create free cluster (M0)
- [ ] Create database user
- [ ] Whitelist IP: `0.0.0.0/0`
- [ ] Get connection string
- [ ] Add to Render environment: `MONGODB_URI=<connection-string>`
- [ ] Restart backend service
- [ ] Test ticket storage

## 📦 Model Hosting

Choose one:

### Option A: Git LFS
- [ ] Install Git LFS
- [ ] Track model file: `git lfs track "*.pt"`
- [ ] Commit and push

### Option B: Google Drive
- [ ] Upload model to Google Drive
- [ ] Get shareable link
- [ ] Update MODEL_DOWNLOAD.md with link
- [ ] Add download script to deployment

### Option C: Hugging Face
- [ ] Create Hugging Face account
- [ ] Upload model to Hub
- [ ] Update model_loader.py to download from Hub

## 🔍 Post-Deployment Testing

- [ ] Frontend loads without errors
- [ ] Can create account
- [ ] Can login
- [ ] Can submit prediction
- [ ] Prediction returns results
- [ ] Results are accurate
- [ ] Dashboard shows statistics (if MongoDB connected)
- [ ] Mobile responsive
- [ ] HTTPS enabled (automatic)

## 🐛 Troubleshooting

**Backend won't start:**
- Check logs in Render dashboard
- Verify all dependencies in requirements.txt
- Check model file exists

**Frontend can't connect:**
- Verify VITE_API_URL is correct
- Check CORS settings in backend
- Test backend health endpoint

**Predictions fail:**
- Check model loaded successfully in logs
- Verify model file integrity
- Test locally first

**Slow cold starts:**
- Normal on free tier (15-30s)
- Consider paid tier or keep-alive service
- Or use smaller model

## 📊 Monitoring

- [ ] Set up UptimeRobot (free) to ping backend every 5 min
- [ ] Monitor Render logs for errors
- [ ] Check Vercel analytics
- [ ] Monitor MongoDB usage (if used)

## ✨ You're Live!

Once all checkboxes are complete, your app is deployed and accessible worldwide! 🎉

Share your deployment:

## 🔄 Updates

To update your deployment:
```bash
git add .
git commit -m "Update: description"
git push
```

Both Render and Vercel will auto-deploy on push! 🚀
